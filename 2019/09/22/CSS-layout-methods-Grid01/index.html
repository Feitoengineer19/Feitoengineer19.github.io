CSS排版方法論-Grid 01：外部容器

Sep 22 2019 CSS

其實，除了 Bootstrap ， CSS3 也有原生的網格系統（Grid System），來看看怎麼使用吧！

Grid 網格系統

網格系統（Grid System）其實歷史悠遠，最早於十六世紀就已經應用在字體設計上，於第二次世界大戰後風行於平面設計中。

所謂的網格，就是由列（Row）與行（Column）縱橫交錯而成的排版系統，通常一個 Row 會切割為 12 個 Column ， Column 與 Column 之間會有間隙（Gutter）區隔彼此，簡單如下圖所示：

CSS 框架 Bootstrap 問世時就定義了專屬的 Grid System ，而後 CSS3 也設計了屬於自己的原生 Grid 屬性，逐漸為主流瀏覽器所支援，詳細可參考 Can I Use。只要確實掌握，未來我們就不需要單純為了 Grid System 而特別載入 Bootstrap 。

以下簡單介紹 CSS3 原生 Grid 屬性中的外部容器及相關語法，請搭配 CodePen DEMO 來閱讀本文。

外部容器（Grid-container）

如同 Bootsrap 、 Flex 需要 container 來包覆內部元素，在使用 CSS3 原生 Grid 時，我們也需要一個外部容器來定義 Grid 的範圍。我習慣將這個 HTML 元素的 class 命名為 grid-container ，並在 CSS 中透過 display 設定 grid 屬性。

grid

功能：將該元素設定為 Grid 外部容器。
語法：display: grid | inline-grid

grid-template-columns

設定外部容器 grid-container 後，接著我們要定義網格中的欄位數量，來排列內部元素 box。

和 Flex 類似，我們在外部容器 grid-container 上定義欄位寬度，定義幾個數值，就表示分割為幾欄位。

功能：設定 Grid 中的欄位寬度（單位：px、%、fr、auto），設定 n 個數值代表有 n 個 Column。
補充：

  1. 單位 fr，類似於 Bootstrap 的 col- ，它會自動幫我們計算 col 的欄寬（占總 fr 的幾分之幾）。
  2. 單位 auto，依據欄位元素的內容自適應寬度，再自動計算該欄位元素所需的寬度。

語法：grid-template-columns: Col1-width Col2-width ...

  • grid-template-columns: 50% 50%（2 欄）
  • grid-template-columns: 20% 30% 50%（3 欄）
  • grid-template-columns: 3fr 2fr 1fr（4 欄，欄寬依序為總欄位數的 1/2 、 1/3 、 1/6）
  • grid-template-columns: auto 2fr 1fr（試著修改 CodePen 玩玩看）
  • grid-template-columns: repeat(4, 1fr)（repeat函式，4 欄，欄寬皆為總欄位數的 1/4）
  • grid-template-columns: repeat(auto-fill, minmax(100px, 1fr)（設定每一個欄位的最小寬度與最大寬度，響應式效果最佳！）

grid-gap

內部元素緊緊靠在一起，看起來不是很舒服，此時，我們可以用 grid-gap 來做出 Gutter。

功能：設定 Grid 的間隙（Gutter）寬度。
語法：grid-gutter: Row-gutter-width Col-gutter-width

  • grid-gutter: 10px 20px（Row-gutter-width:10px ＆ Col-gutter-width:20px）
  • grid-gutter: 20px（行、欄 Gutter 寬度都是20px）

補充：也可以單獨設定 grid-column-gap 或 grid-row-gap。

grid-template-rows

除了定義欄寬，我們也可以用 grid-template-rows 定義列寬，用法和 grid-template-columns 相同（但比較少用？）。

功能：設定 Grid 中的列位寬度（單位：px、%、fr、auto），設定 n 個數值代表有 n 個 Row。
語法：grid-template-rows: Row1-width Row2-width ...

  • grid-template-rows: 1fr 3fr 2fr（3 欄）

補充：

  1. 若設定的列位數大於內部元素數量，Grid 還是會設定列位數，Grid 範圍會變大。
  2. 若設定的列位數小於內部元素數量，Grid 還是會設定列位數，Grid 範圍會變小。
